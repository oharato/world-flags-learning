# 基本設計書 - 国旗学習アプリ

## 1. 設計概要

### 1.1. 目的
本ドキュメントは、「国旗学習アプリ」のシステム設計を定義するものである。要件定義書に基づき、機能、データ構造、システム構成を明確にする。

### 1.2. システム構成
*   **フロントエンド**: React (SPA: Single Page Application)
    *   UIの描画、ユーザー操作の受付、バックエンドAPIとの通信を担当する。
*   **バックエンド**: Node.js / Express
    *   ランキングデータの永続化と提供のためのAPIサーバーとして機能する。
*   **データベース**: SQLite
    *   ランキングデータを保存する軽量なデータベース。サーバー内にファイルとして保持する。

## 2. 機能詳細設計

### 2.1. データ構造
#### 2.1.1. 国データ
アプリケーション内で使用する国に関する情報は、フロントエンド側にJSONファイルとして保持する。
`countries.json`
```json
[
  {
    "id": "jpn",
    "name": "日本",
    "capital": "東京",
    "continent": "アジア",
    "flag_image_url": "/flags/jpn.svg",
    "map_image_url": "/maps/jpn.svg",
    "description": "日本の国旗は「日章旗」と呼ばれ、太陽を象徴しています。...",
    "summary": "人口は約1.2億人、公用語は日本語です。"
  },
  // ... 他の国のデータ
]
```

#### 2.1.2. ランキングデータ (DBテーブル)
`ranking` テーブル
| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | INTEGER | 主キー (自動採番) |
| `nickname` | TEXT | ユーザーニックネーム |
| `score` | INTEGER | スコア |
| `created_at` | TEXT | 登録日時 (ISO 8601形式) |

### 2.2. 機能ごとのロジック
#### 2.2.1. クイズモード
*   **問題生成ロジック**:
    *   出題する10問は、`countries.json` からランダムに重複なく選出する。
    *   各問題の選択肢（4択）は、正解の1つと、不正解の3つ（`countries.json`からランダムに選出）で構成する。不正解の選択肢は、正解と重複しないようにする。
*   **スコア計算**:
    *   要件定義通り `スコア = (正解数 * 1000) - (回答時間[秒] * 10)` とする。
    *   時間はクイズ開始から最終問題の回答完了までの秒単位の時間とする。

#### 2.2.2. ランキング機能
*   **登録**: クイズ完了後、結果画面でニックネームと算出されたスコアをバックエンドAPIに送信し、DBに登録する。
*   **表示**: ランキング画面表示時、バックエンドAPIから上位100件のランキングデータを取得して表示する。
*   **順位付け**: `score` の降順（高い順）で順位を決定する。スコアが同点の場合は、`created_at` の昇順（登録が早い順）で上位とする。

#### 2.2.3. 学習モード
*   **表示ロジック**:
    *   `countries.json` のデータをリストとして読み込む。
    *   「次へ」「前へ」ボタンで、リストのインデックスを移動し、対応する国の情報を表示する。
    *   カードの表面には国旗画像、裏面には国名やその他の詳細情報を表示する。クリックまたはタップでカードが回転するアニメーションを実装する。
